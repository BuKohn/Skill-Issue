# Skill Issue

**Skill Issue** — веб-проект, платформа для обмена навыками. Пользователи могут создавать объявления о своих навыках, находить партнёров для обучения и общаться через текстовый чат. Платформа также включает систему профилей с рейтингами и отзывами.


## Цели проекта
- Создать удобную платформу для обмена навыками.
- Обеспечить возможность поиска и фильтрации объявлений.
- Реализовать систему профилей, рейтингов и отзывов.
- Обеспечить стабильную работу веб-сайта.

---

## Функциональность
- Регистрация и авторизация пользователей.
- Создание, редактирование и просмотр объявлений о навыках.
- Лента объявлений с поиском и базовыми фильтрами.
- Текстовый чат для первичной связи между пользователями.
- Профили пользователей с информацией о навыках и рейтингом.
- Система отзывов и рейтингов, автоматически обновляющая рейтинг профиля.

## API Документация (Swagger)

Проект использует Swagger (OpenAPI) для автоматической генерации интерактивной документации API. Swagger позволяет разработчикам описывать структуру API и генерировать интерактивную документацию, клиентские библиотеки и серверные модули.

### Установка зависимостей

Перед использованием Swagger документации убедитесь, что установлены все необходимые зависимости:

```bash
pip install -r requirements.txt
```

### Доступ к Swagger документации

После запуска сервера разработки, Swagger документация доступна по адресу:

**Swagger UI** (интерактивный интерфейс):
```
http://127.0.0.1:8000/swagger/
```

Swagger UI предоставляет удобный веб-интерфейс для просмотра и тестирования всех API эндпойнтов.

### Использование Swagger UI

1. Откройте браузер и перейдите по адресу `http://127.0.0.1:8000/swagger/`
2. Вы увидите список всех доступных API эндпойнтов, сгруппированных по категориям:
   - **Аутентификация** - регистрация и вход пользователей
   - **Пользователи** - информация о текущем пользователе
   - **Профили** - профили пользователей
   - **Руководства** - создание и управление руководствами (гайдами)
   - **Отзывы** - отзывы на руководства
   - **Комментарии** - комментарии к объявлениям
   - **Поиск** - поиск по платформе

3. Для тестирования API:
   - Нажмите на любой эндпойнт, чтобы развернуть его описание
   - Нажмите кнопку **"Try it out"**
   - Заполните необходимые параметры запроса
   - Нажмите **"Execute"** для отправки запроса
   - Просмотрите ответ сервера в разделе **"Responses"**

### Авторизация в Swagger

Для тестирования защищенных эндпоинтов (требующих авторизации):

1. Сначала выполните запрос на `/api/login/` через Swagger UI
2. После успешной авторизации, сессия будет сохранена в браузере
3. Теперь вы можете использовать защищенные эндпоинты

### Основные API эндпоинты

#### Аутентификация
- `POST /api/register/` - Регистрация нового пользователя
- `POST /api/login/` - Авторизация пользователя
- `GET /api/me/` - Получить информацию о текущем пользователе

#### Профили
- `GET /api/profile/<username>/` - Получить профиль пользователя

#### Руководства
- `GET /api/guides/` - Список всех руководств
- `POST /api/guides/` - Создать новое руководство
- `GET /api/guides/{id}/` - Получить руководство по ID
- `PUT /api/guides/{id}/` - Обновить руководство
- `DELETE /api/guides/{id}/` - Удалить руководство
- `POST /api/guides/{guide_id}/rate/` - Оценить руководство

#### Объявления
- `GET /api/announcements/` - Список всех объявлений
- `POST /api/announcements/` - Создать новое объявление
- `GET /api/announcements/{id}/` - Получить объявление по ID
- `PUT /api/announcements/{id}/` - Обновить объявление
- `DELETE /api/announcements/{id}/` - Удалить объявление

#### Отзывы
- `POST /api/reviews/create/` - Создать отзыв на руководство
- `PUT /api/reviews/{id}/update/` - Обновить отзыв
- `DELETE /api/reviews/{id}/delete/` - Удалить отзыв

#### Комментарии
- `POST /api/announcements/comments/create/` - Создать комментарий к объявлению
- `PUT /api/announcements/comments/{id}/update/` - Обновить комментарий
- `DELETE /api/announcements/comments/{id}/delete/` - Удалить комментарий

#### Поиск
- `GET /api/search/all/` - Получить все элементы для поиска
- `GET /api/search/?q=<query>` - Поиск по запросу
